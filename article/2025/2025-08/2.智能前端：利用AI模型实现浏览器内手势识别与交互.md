当前，前端也能够通过集成AI模型实现浏览器内的手势识别与交互功能，Google推出的一个关于机器学习的开源免费的项目**Mediapipe**，可精准检测并处理人脸特征、人体姿态、手部关键点乃至发际线轮廓等多种视觉数据。 

## Mediapipe

### 介绍

MediaPipe 解决方案提供了一套库和工具，可快速在应用中使用人工智能 (AI) 和机器学习 (ML) 技术

![image-20250829191612943](/Users/songyao/Desktop/songyao/fe-nanjiu/article/2025/2025-08/images/2.png)

- **MediaPipe Tasks**：提供跨平台 API 和库。
- **MediaPipe model**：提供每个解决方案搭配使用的预训练AI大模型。
- **MediaPipe Model Maker**：用于使用自己的数据自定义模型。
- **MediaPipe Studio**：在浏览器中直观呈现、评估和对比解决方案。

### 解决方案

MediaPipe 提供了多个解决方案并且适用于多个平台。每个解决方案都包含一个或多个模型，并且还可以为某些解决方案自定义模型。

### 演示

#### 安装

```shell
pnpm add @mediapipe/tasks-vision
```

#### 创建任务

MediaPipe 手势识别器任务需要使用与此任务兼容的训练模型

```js
// 加载指定版本的MediaPipe视觉任务WebAssembly模块
const vision = await FilesetResolver.forVisionTasks("https://cdn.jsdelivr.net/npm/@mediapipe/tasks-vision@0.10.3/wasm");
try {
      gestureRecognizer = await GestureRecognizer.createFromOptions(vision, {
          baseOptions: {
              modelAssetPath: 'https://storage.googleapis.com/mediapipe-models/gesture_recognizer/gesture_recognizer/float16/1/gesture_recognizer.task',
          },
          numHands: 1,  // 识别手的数量
          runningMode: 'VIDEO' // 视频模式
      });
  } catch (error) {
      console.log('手势识别器加载失败', error);
  }
```

这里`modelAssetPath`指的就是识别手势的大模型路径，文件有点大加载会比较耗时，不过我们可以将该二进制文件下载到本地使用，可以提升下载速度。

#### 读取视频流

这里可以调用系统摄像头，获取到视频流添加到video元素中

```js
// 打开摄像头
navigator.mediaDevices.getUserMedia({ video: true }).then(stream => {
  // 视频流添加到video元素中
  tracks = stream.getTracks();
  video.srcObject = stream;
  // 绑定视频加载完成事件,开始识别视频中的手势
  video.addEventListener('loadeddata', () => {
    predictWebcam();
  });

});
```

**从 Chrome 47 开始，getUserMedia() 请求只允许来自安全来源： HTTPS 或 localhost**

#### 任务输入

从上面`runningMode`参数可以看到，我们指定的是`VIDEO`表示输入的内容是视频，实际上它支持的输入内容有以下三种：

- IMAGE：静态图片
- VIDEO：解码后的视频帧
- LIVE_STREAM：输入数据（例如来自摄像头）的直播模式。在此模式下，必须调用 resultListener 以设置监听器以异步接收结果。

```js
const predictWebcam = () => {
  let results = {};
  // 获取video元素
  const video = document.getElementById('webcam') as HTMLVideoElement;
  // 如果视频的时间发生变化,则识别视频中的手势
  if (video.currentTime !== lastVideoTime) {
    // 替换上次识别视频手势的时间
    lastVideoTime = video.currentTime;
    // 使用recognizeForVideo对视频进行推理，该任务会处理数据，尝试识别手势，然后返回结果。
    results = gestureRecognizer?.recognizeForVideo(video, video.currentTime);
  }
}
```

手势识别器使用 `recognize()`（运行模式为 `'image'`）和 `recognizeForVideo()`（运行模式为 `'video'`）方法触发推理。该任务会处理数据，尝试识别手势，然后报告结果。

#### 任务输出

手势识别器会输出以下结果：

![image-20250901203728535](/Users/songyao/Desktop/songyao/fe-nanjiu/article/2025/2025-08/images/3-1.png)

- **gesture：**手势类别

- **handednesses：**检测到的手的惯用手

- **landmarks：**检测到的手的地标（以图片坐标表示）

- **worldLandmarks：**以世界坐标表示的检测到的手的特征点，每个地标均由 `x`、`y` 和 `z` 组成，表示以米为单位的真实 3D 坐标，其原点位于手的几何中心。

`gesture`包含7种常见的手势类型，如不在这7种手势范围内，则会返回Unrecognized gesture，表示无法识别的手势：

```
0 - Unrecognized gesture, label: Unknown 【未知】
1 - Closed fist, label: Closed_Fist 【握拳】
2 - Open palm, label: Open_Palm  【张开手掌】
3 - Pointing up, label: Pointing_Up  【指向上】
4 - Thumbs down, label: Thumb_Down  【拇指朝下】
5 - Thumbs up, label: Thumb_Up  【竖起大拇指】
6 - Victory, label: Victory  【胜利】
7 - Love, label: ILoveYou 【比心】
```

`landmarks`、`worldLandmarks`包含检测到的手部区域内 21 个手指关节坐标的关键点定位。该模型是基于约 3 万张真实图片以及叠加在各种背景上的多种渲染合成手部模型进行训练的。

    color: "#FF0000",
    lineWidth: 1,
    radius: 4
  
  });
};

## 手势识别画板

深入理解MediaPipe后，开发者可以实现以下曾经难以想象的功能：

- 基于手势识别的实时绘制

- 基于手势识别的画布清除
- 自定义手势触发保存功能

整套交互流程完全脱离传统鼠标操作，重新定义人机交互方式。

### 定义绘制手势

在开始绘制前，我们需要预先定义一个特定的手势作为【握笔✍️】动作（如🤏捏合动作）。只有当系统识别到用户做出该预设手势并在屏幕前移动时，才会触发绘制功能，其他手势则不会产生绘制效果，避免误操作。

```js
const thumbTip = landmarks[4]; // 大拇指尖
const indexFingerTip = landmarks[8]; // 食指尖

const dx = (thumbTip.x - indexFingerTip.x) * width; // 计算x轴距离
const dy = (thumbTip.y - indexFingerTip.y) * height; // 计算y轴距离

const connected = dx < 20 && dy < 20; // 判断是否近似贴合
if (connected) {
  const x = (1 - indexFingerTip.x) * width; // 计算x坐标
  const y = indexFingerTip.y * height; // 计算y坐标
  writeText(penCanvasCtx, x, y); // 实时绘制
} else {
  prevX = prevY = 0;
}
```

### 实时绘制

```js
// 写字
const SMOOTHING_FACTOR = 0.6; // 平滑因子，用来解决绘制抖动，避免线条断断续续
const writeText = (ctx: CanvasRenderingContext2D | null, x: number, y: number) => {
  if (!ctx) return;
  if (!prevX || !prevY) {
      prevX = x;
      prevY = y;
    }
    const smoothedX = prevX + SMOOTHING_FACTOR * (x - prevX); // 平滑处理x坐标
    const smoothedY = prevY + SMOOTHING_FACTOR * (y - prevY); // 平滑处理y坐标
    ctx.lineWidth = 5; // 画笔宽度
    ctx.moveTo(prevX, prevY); // 画笔跟随
    ctx.lineTo(smoothedX, smoothedY); // 画笔移动
    ctx.strokeStyle = "#191919"; // 画笔颜色
    ctx.stroke(); // 画笔绘制
    ctx.save(); // 保存当前状态

    prevX = smoothedX;
    prevY = smoothedY;
}
```

### 画布清除

这里同样需要定义一个手势用于擦除操作，为了方便处理，我们可以直接使用现成的【Open_Palm】张开手掌手势

```js
if(categoryName === enumGesture.Open_Palm) {
    const landmarks = results.landmarks[0];
    // 计算手掌的中心点，将9当成近似中心点
    const palmCenterX = (1 - landmarks[9].x) * width;
    const palmCenterY = landmarks[9].y * height;
    // 计算手掌长宽
    const handWidth = (1 - landmarks[0].x) * width - (1 - landmarks[5].x) * width;
    const handHeight = landmarks[0].y * height - landmarks[5].y * height;
    // 以手掌中心为中心点，手掌区域清除画布
    clearCanvas(penCanvasCtx, palmCenterX - handWidth / 2, palmCenterY - handHeight / 2, handWidth, handHeight);
  }

// 根据手掌位置擦除画布
const clearCanvas = (ctx: CanvasRenderingContext2D | null, x: number, y: number, width: number, height: number) => {
  if (!ctx) return;

  ctx.clearRect(x, y, width, height);
  ctx.restore();
  ctx.save();
};
```

```js
// 截屏
const takeScreenshot = () => {
  const penCanvasElement = document.getElementById('penCanvas') as HTMLCanvasElement; // 获取手写区域的canvas
  const canvas = document.createElement('canvas');
  canvas.width = penCanvasElement.width;
  canvas.height = penCanvasElement.height;
  const ctx = canvas.getContext('2d');
  if (ctx) {
    ctx.drawImage(penCanvasElement, 0, 0, canvas.width, canvas.height);
    const dataURL = canvas.toDataURL('image/png');
    // 下载图片
    const link = document.createElement('a');
    link.href = dataURL;
    link.download = 'screenshot.png';
    document.body.appendChild(link);
    link.click();
    document.body.removeChild(link);
  }
}
```
