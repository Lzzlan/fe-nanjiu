## 前言

在 Vue3 中，`computed` 计算属性被广泛使用，但很多开发者误以为它**只能用于读取数据**，无法修改。其实，`computed` 支持**两种用法**：

1. **只读计算属性**（默认用法）
2. **可写计算属性**（支持 `get` + `set`）

本文将深入解析这两种用法的区别，并探讨其底层实现原理，帮助你更灵活地使用 `computed`！

## computed

> 接受一个 getter 函数，返回一个只读的响应式 ref 对象。该 ref 通过 `.value` 暴露 getter 函数的返回值。它也可以接受一个带有 `get` 和 `set` 函数的对象来创建一个可写的 ref 对象。

**computed的核心优势在于其高效的缓存机制：只有当依赖的响应式数据发生变更时，才会重新计算值；否则直接返回缓存结果，避免不必要的计算开销。**

### 函数式

这是大家最常用的方式，接受一个函数返回一个只读的响应式ref对象

```vue
<template>
    <div>
        <input type="text" v-model="province" placeholder="请输入省份/直辖市" />
        <input type="text" v-model="city" placeholder="请输入市/区" />

        <p>地址: {{ address }}</p>
    </div>
</template>

<script setup lang="ts">
import { ref, computed, isRef, isReadonly } from 'vue';

const province = ref('');
const city = ref('');

const address = computed(() => {
    return `${province.value} - ${city.value}`;
});
console.log('address:', address);
console.log('isRef:', isRef(address));
console.log('isReadonly:', isReadonly(address));
</script>
```

### 对象式

这种方式很多同学都没怎么用过，所以会误以为computed值一定是无法修改的，对象的方式可以接收`get`和`set`两个函数来分别对应**computed**的`读`和`写`

```js
const address = computed({
    get() {
        return `${province.value} - ${city.value}`;
    },
    set(newValue) {
        const [newProvince, newCity] = newValue.split('-');
        province.value = newProvince || '';
        city.value = newCity || '';
    }
});

address.value = `上海市-浦东新区`;
console.log('address:', address);
console.log('isRef:', isRef(address));
console.log('isReadonly:', isReadonly(address));
```

所以这种方式的`computed`就是可读可写的，后面遇到这种问题千万别再说computed是只读的了。

**修改computed**

```js
const address = computed({
    get() {
        return `${province.value} - ${city.value}`;
    },
    set(newValue) {
        const [newProvince, newCity] = newValue.split('-');
        province.value = newProvince || '';
        city.value = newCity || '';
    }
});

address.value = `上海市-浦东新区`;
const updateComputed = () => {
    address.value = `北京市-朝阳区`;
}
```
